Data Version Control as a Software

Data Version Control is a method of working with datasets, similar to version control systems used in traditional software development. However, it has been enhanced to better support data processing and collaboration in the context of data analytics, research, and other types of data analysis. It includes specific features and configurations tailored to enable work with massive datasets and data lakes.

Distributors of DVC solutions:

  • lakeFS - lakeFS is a scalable data version control platform that provides a Git-like experience for object-storage based data lakes. It is mentioned as a leading solution for managing data version control.

  • DVC - DVC is a version control system designed to work with version-controlled systems like Git. It allows users to upload data to a remote storage service and generate metadata files for version control. However, it is mentioned that DVC may not be the best choice for relational database users and may have limitations when operating on a large scale.

  • The Nessie Project - The Nessie Project is an open-source project that provides control and consistency around data. It draws inspiration from GitHub and applies development methodologies and concepts to data versioning. Nessie enables data engineers and analysts to update, restructure, and repair datasets while maintaining a consistent version of truth.
